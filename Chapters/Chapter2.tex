\section{The MDP Framework}
\label{sec:mdpframework}

The formalization of \acrshort{rl} problems within the \acrshort{mdp} framework can best be described using the agent-environment interface.

\begin{figure}[h]
	\centering
	\includegraphics[width=0.6\textwidth]{Figures/AgentEnvironmentInterface.pdf}
	\caption{Own representation based on the agent-environment interaction\\ in a Markov decision process \citep[Figure~3.1]{suttonReinforcementLearningIntroduction2020}.}
	\label{fig:yourlabel}
\end{figure}

The agent interacts with the environment in order to achieve a goal. Following the reward hypothesis \citep[p.~53]{suttonReinforcementLearningIntroduction2020}, this goal can be expressed as the maximization of a cumulative reward signal, formalized in the next section as the expected discounted return. At each timestep the agent observes a representation of the environment's state, selects an action, receives a reward, and the environment transitions to a new state. When this interaction satisfies the Markov property, meaning the next state and reward depend only on the current state and action, the problem can be formalized as an \acrshort{mdp}. Chapter~\ref{chap:pong} formalizes the game of Pong within this framework. A recurring theme in the chapters on \acrshort{dqn} and \acrshort{a2c} is how each algorithm navigates the fundamental tension between exploiting actions that have returned high reward so far and exploring untried actions that may return better outcomes. When the state space $\mathcal{S}$, action space $\mathcal{A}$, and reward set $\mathcal{R}$ all have a finite number of elements, the \acrshort{mdp} is said to be finite. In this case, given a specific preceding state $s$ and action $a$, the probability of transitioning to any particular successor state $s^{\prime} \in \mathcal{S}$ and receiving a specific reward $r \in \mathcal{R}$ at time $t$ is given by:
\begin{equation}\label{eq:finite_mdp}
	p(s^{\prime}, r\;|\;s, a) \doteq Pr\{S_t = s^{\prime}, R_t = r\;| \;S_{t-1} = s, A_{t-1} = a\},
\end{equation}
for all $s^{\prime}, s \in \mathcal{S}$, $r \in \mathcal{R}$, and $a \in \mathcal{A}(s)$. The probability distribution $p$ completely describes the  dynamics of the \acrshort{mdp} for each choice of $s$ and $a$ such that:
\begin{equation}
	\sum_{s^{\prime} \in \mathcal{S}}\;\sum_{r \in \mathcal{R}}\;p(s^{\prime}, r\;|\;s, a) = 1,\;\;\text{for all}\; s \in \mathcal{S}, a \in \mathcal{A}(s)
\end{equation}

The \acrshort{mdp} formalizes what the agent interacts with. The next section introduces the machinery for evaluating how well the agent learns.

\section{Value Functions and the Bellman Equations}
\label{sec:valuefunctionbellman}
Value functions, used in almost all \acrshort{rl} algorithms, estimate how good it is for the agent to be in a given state, or to take a given action in a given state \citep[Chapter~3]{suttonReinforcementLearningIntroduction2020}. Here, `good' is defined as the cumulative reward the agent can expect to receive from that point onward. This expected cumulative reward is formalized through the discounted return:
\begin{equation}\label{eq:discounted_return}
	G_t \doteq R_{t+1} + \gamma R_{t+2} + \gamma^{2} R_{t+3} + \dots = \sum_{k=0}^{\infty} \gamma^{k} R_{t+k+1},
\end{equation}
where $\gamma$ is a parameter, $0 \leq \gamma \leq 1$, called the discount rate. The value of a reward is discounted by a factor of $\gamma^{k-1}$ when it arrives k time steps later rather than immediately. The discount rate determines the present value of future rewards. With $\gamma$ close to zero, the agent is near-sighted and weighs immediate rewards heavily, while a value close to one makes the agent far-sighted and accounts for rewards far into the future.

The expected return depends on how the agent selects actions. The mapping between the state and the probability of selecting each action is formally called a policy and is defined as $\pi{\left(a|s\right)}$, where $a \in \mathcal{A}(s)$ for each $s \in \mathcal{S}$. When starting from a state $s$, its value function, $	v_\pi\left(s\right)$, when following the policy $\pi$, is the expected return:
\begin{equation}\label{eq:state_value_function}
	v_\pi\left(s\right) \doteq \mathbb{E}_\pi\left[G_t\,|\,S_t = s\right] = \mathbb{E}_\pi\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1}\,|\,S_t = s \right],\;\,\text{for all}\, s \in \mathcal{S},
\end{equation}
where $\mathbb{E}_\pi[\cdot]$ represents the expected value of a random variable when the policy $\pi$ is followed, and $t$ is any time step. The function $v_\pi\left(s\right)$ is called the state-value function. An equivalent function exists for when taking action $a$ in state $s$ following policy $\pi$ and is called the action-value function denoted as:
\begin{equation}\label{eq:action_value_function}
	q_\pi\left(s, a\right) \doteq \mathbb{E}_\pi\left[G_t\,|\,S_t = s, A_t = a\right] = \mathbb{E}_\pi\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1}\,|\,S_t = s, A_t = a\right],\;\,\text{for all}\, s \in \mathcal{S},\,\text{and}\, a \in \mathcal{A}(s).
\end{equation}


A fundamental property of the return is that it can be written recursively, which leads directly to the Bellman equations \citep{bellmanDynamicProgramming1957}. The returns in \eqref{eq:discounted_return} at successive time steps are related to each other:
\begin{align}\label{eq:recursive_return}
	G_t &\doteq R_{t+1} + \gamma R_{t+2} + \gamma^{2} R_{t+3} + \gamma^{3} R_{t+4} + \dots \nonumber \\
	&=  R_{t+1} + \gamma \left(R_{t+2} + \gamma R_{t+3} + \gamma^{2} R_{t+4} + \dots \right) \nonumber \\
	&= R_{t+1} + \gamma G_{t+1}
\end{align}
and is used to derive the Bellman equation for $v_\pi$:
\begin{align*}
	v_\pi\left(s\right) &\doteq \mathbb{E}_\pi\left[G_t\,|\,S_t = s\right] \tag{from \ref{eq:state_value_function}}\\
	&= \mathbb{E}_\pi\left[R_{t+1} + \gamma G_{t+1}\,|\,S_t = s\right]. \tag{by \ref{eq:recursive_return}}
\end{align*}

Expanding the expectation over the \acrshort{mdp} dynamics and applying the Markov property gives:
\begin{align}
	v_\pi\left(s\right) &= \sum_{a} \pi(a|s) \sum_{s^{\prime}} \sum_{r}p(s^{\prime}, r\;|\;s, a)\,\bigg[r + \gamma \mathbb{E}_\pi\left[G_{t+1}\,|\,S_{t+1} = s^{\prime}\right] \bigg] \nonumber \\
	&= \sum_{a} \pi(a|s) \sum_{s^{\prime}, r} p(s^{\prime}, r\;|\;s, a) \left[r + \gamma v_\pi(s^{\prime})\right],\;\;\text{for all}\;s,s^{\prime} \in \mathcal{S}, a \in \mathcal{A}(s), \text{and}\;r \in \mathcal{R},
\end{align}
where the last two sum notations are merged together. The expected value $\mathbb{E}$ is broken down into the sum over all the values of $a$, $s^\prime$ and $r$ weighted by their combined probability $\pi(a|s)\,p(s^{\prime}, r\;|\;s, a)$.

By analogous reasoning, the Bellman equation for the $q_\pi$ is:
\begin{equation}
	q_\pi(s, a) = \sum_{s', r} p(s', r \mid s, a) \left[ r + \gamma \sum_{a'} \pi(a' \mid s') \, q_\pi(s', a') \right],
\end{equation}
for all $s,s^{\prime} \in \mathcal{S}$, $a \in \mathcal{A}(s)$, $a^{\prime} \in \mathcal{A}(s^{\prime})$ and $r \in \mathcal{R}$.

Both the state-value and action-value Bellman equations can be written in terms of an optimal policy, $\pi_*$, which is a policy better than or equal to all other policies in every state. The value functions using $\pi_*$ satisfy the Bellman optimality equations denoted as $v_*(s)$ and $q_*(s,a)$ correspondingly:
\begin{equation}
	v_*(s) = \max_{a} \sum_{s^{\prime}, r} p(s^{\prime}, r\;|\;s, a) \left[r + \gamma v_*(s^{\prime})\right],
\end{equation}
\begin{equation}
	q_*(s,a) = \sum_{s^{\prime}, r} p(s^{\prime}, r\;|\;s, a) \left[r + \gamma \max_{a^{\prime}}q_*(s^{\prime}, a^{\prime})\right].
\end{equation}

\acrshort{dqn} targets the Bellman optimality equation for the action-value function $q_*(s,a)$, while \acrshort{a2c}'s critic targets the Bellman expectation equation for the state-value function $v_\pi\left(s\right)$.

\section{Policy Optimization}
\label{sec:policyoptimization}
Sections~\ref{sec:mdpframework} and \ref{sec:valuefunctionbellman} established the \acrshort{mdp} and defined what an optimal policy is. The optimal policy $\pi_*$ maximizes expected return from every state. Knowing what $\pi_*$ is, however, is not the same as knowing how to find it. For any problem of realistic scale, exact solution is computationally infeasible \citep[pp.~67, 69]{suttonReinforcementLearningIntroduction2020}. Finding $\pi_*$ therefore requires computational strategies rather than direct solution, and two such strategies form the subject of this section.

The first strategy takes the value-based route. Using the Bellman optimality equation for $q_*$, the optimal policy can be obtained as $\pi_*(s) = \argmax_{a} q_*(s,a)$. The learning problem can then approximate $q_*$ to determine the optimal policy implicitly. This is the path Chapter~\ref{chap:dqn} develops into \acrshort{dqn}.

The second strategy takes the policy-based route where the dependency is inverted. Instead of learning a value function and deriving a policy based on it, the policy is directly parameterized as $\pi(a|s, \theta) = Pr\{A_t=a|S_t=s, \theta_t=\theta\}$ for the probability that action $a$ is taken at time $t$ when the environment is at state $s$ at time $t$ with parameter $\theta$. A value function can still be used but not for action selection but rather for learning the policy parameter $\theta$. This is the path Chapter~\ref{chap:a2c} develops from REINFORCE to \acrshort{a2c}.

The value-based and policy-based routes differ substantively. They learn different objects, represent the policy implicitly and deterministically versus explicitly and stochastically, handle exploration through $\epsilon$-greedy action selection versus policy entropy, and naturally target discrete versus continuous action spaces. Chapter~\ref{chap:comparativeanalysis} examines empirically how these differences affect sample efficiency, stability, and problem suitability.
